# Versioned Deserialization для Rapira

## Context

Данные хранятся в БД. Миграции будут применяться, но в короткий промежуток нужна возможность читать старые данные (без новых полей) новым кодом. Версия схемы хранится **внешне** (в метаданных БД), не в сериализованных данных. Нужна только обратная совместимость (новый код + старые данные).

## Подход

Добавить метод `from_slice_versioned(slice, version)` в трейт `Rapira` с default-реализацией, которая делегирует к `from_slice`. Для versioned-структур derive-макрос генерирует override, который условно читает поля на основе переданной версии. Коллекции (Vec, BTreeMap, Box, Option и т.д.) получают override для проброса версии во вложенные типы.

**Сериализация не меняется** — `convert_to_bytes`, `size`, `try_convert_to_bytes` всегда пишут все поля (текущая версия). Изменения только в десериализации.

**Без `since` атрибутов** — код генерируется точно как раньше (from_slice_versioned просто вызовет from_slice по default-impl).

```rust
fn from_slice_versioned(slice: &mut &[u8], version: u8) -> rapira::Result<Self> {
    // v1 поля — читаются безусловно
    let name = <String as rapira::Rapira>::from_slice_versioned(slice, version)?;
    let age = <u32 as rapira::Rapira>::from_slice_versioned(slice, version)?;
    // v2+ поля — условно
    let email = if version >= 2 {
        <Option<String> as rapira::Rapira>::from_slice_versioned(slice, version)?
    } else {
        Default::default()
    };
    Ok(Self { name, age, email })
}
```

**`from_slice` не меняется** — всегда читает все поля (текущая версия).
**`convert_to_bytes`, `size` не меняются** — всегда пишут все поля.
**`STATIC_SIZE`, `MIN_SIZE` не меняются** — считаются по всем полям.

### 10. Валидация (compile-time errors в макросе)

- `#[rapira(since = N)]` без `#[rapira(version = M)]` на структуре → ошибка компиляции
- `since > version` → ошибка
- `since = 0` → ошибка
- `since` + `skip` → ошибка

### 11. Тесты — `rapira/tests/all.rs`

```rust
#[derive(Debug, PartialEq, Rapira)]
#[rapira(version = 2)]
struct UserV2 {
    name: String,
    age: u32,
    #[rapira(since = 2)]
    email: Option<String>,
}

#[test]
fn test_versioned_roundtrip() {
    let user = UserV2 { name: "Alice".into(), age: 30, email: Some("a@b.com".into()) };
    let bytes = rapira::serialize(&user);
    let deser: UserV2 = rapira::deserialize(&bytes).unwrap();
    assert_eq!(user, deser);
}

#[test]
fn test_versioned_backward_compat() {
    // Симулируем v1 данные (name + age, без email)
    let mut v1_bytes = Vec::new();
    rapira::extend_vec(&"Alice".to_string(), &mut v1_bytes);
    rapira::extend_vec(&30u32, &mut v1_bytes);

    let user: UserV2 = rapira::deserialize_versioned(&v1_bytes, 1).unwrap();
    assert_eq!(user.name, "Alice");
    assert_eq!(user.age, 30);
    assert_eq!(user.email, None); // Default
}

#[test]
fn test_versioned_in_vec() {
    // Vec<UserV2> с v1 данными
    let mut v1_bytes = Vec::new();
    rapira::extend_vec(&2u32, &mut v1_bytes); // count = 2
    rapira::extend_vec(&"Alice".to_string(), &mut v1_bytes);
    rapira::extend_vec(&30u32, &mut v1_bytes);
    rapira::extend_vec(&"Bob".to_string(), &mut v1_bytes);
    rapira::extend_vec(&25u32, &mut v1_bytes);

    let users: Vec<UserV2> = rapira::deserialize_versioned(&v1_bytes, 1).unwrap();
    assert_eq!(users.len(), 2);
    assert_eq!(users[0].email, None);
    assert_eq!(users[1].email, None);
}
```

## Что НЕ меняется

- Wire format (формат байтов) — полностью обратно совместим
- Сериализация (`convert_to_bytes`, `size`) — всегда пишет все поля
- `from_slice` — всегда читает все поля (как раньше)
- Структуры без `#[rapira(version)]` — код идентичен текущему
- Примитивные типы — default-impl `from_slice_versioned` вызывает `from_slice`
